This skill enables elite-level CV template and theme design for the Nexus CV Generator. It covers all aspects of document design including layouts, typography systems, color palettes, and ATS optimization.

The agent must understand that CV templates serve dual purposes: human readers (recruiters, hiring managers) and machine readers (ATS systems). Both audiences require careful consideration.

## Design Philosophy

### Purpose-Driven Design
- **Human Readability**: Clean typography, clear hierarchy, professional appearance
- **ATS Compatibility**: Parseable structures, standard headings, no complex layouts
- **Brand Identity**: Consistent visual language that represents the candidate professionally
- **Versatility**: Templates that work across industries and experience levels

### Document Types
- **Resume**: Concise (1-2 pages), achievement-focused, tailored to specific roles
- **CV**: Comprehensive (2+ pages), complete career history, academic/technical
- **Cover Letter**: Personalized introduction, role-specific messaging, call-to-action

## 1. Layout Systems (Plural)

### 1.1 Document Grids
- **Single Column**: Traditional vertical flow, easy reading, ATS-friendly
- **Two Column**: Contact/education on left, experience on right, efficient space use
- **Modern Asymmetric**: Creative balance, distinctive but professional
- **Sidebar Layout**: Persistent sidebar for contact/skills, main content area

### 1.2 Section Structures (Plural)
- **Header Section**: Name, title, contact, social links, location
- **Summary/Profile**: Professional summary, career highlights, value proposition
- **Experience Section**: Roles, companies, dates, achievements, bullet points
- **Education Section**: Degrees, institutions, dates, honors, relevant coursework
- **Skills Section**: Technical skills, soft skills, languages, certifications
- **Projects Section**: Notable projects, technologies used, outcomes
- **Awards/Achievements Section**: Recognition, metrics, impact

### 1.3 Spacing Systems (Plural)
- **Margins**: Standard 0.5-1 inch margins. 0.75 inch preferred.
- **Section Spacing**: 1.5-2x body text spacing between sections
- **Content Spacing**: 0.5-1x line height within sections
- **Consistency**: Use uniform spacing throughout. Never mix multiple spacing values.

### 1.4 Page Layouts
- **A4 (International)**: 210mm x 297mm, standard for most countries
- **Letter (US)**: 8.5" x 11", standard US/Canada
- **Proportions**: Maintain aspect ratios, avoid awkward breaks
- **Widows/Orphans**: Control paragraph breaks, avoid single lines at page bottoms

## 2. Typography Systems (Plural)

### 2.1 Font Categories (Plural)
- **Serif**: Traditional, authoritative, professional. Times New Roman, Georgia, Garamond, Cambria.
- **Sans-Serif**: Modern, clean, readable. Arial, Calibri, Helvetica, Roboto.
- **Monospace**: Technical/resumes, code-related roles. Use sparingly.

### 2.2 Type Hierarchy (Plural)
- **Name/Header**: 14-18pt, bold, potentially larger than body
- **Section Headers**: 11-12pt, ALL CAPS or Title Case, clear distinction
- **Body Text**: 10-11pt, readable size for recruiters
- **Metadata**: 9-10pt, secondary information

### 2.3 Type Scales
- **Modular Scale**: Use 1.2-1.25 ratio between levels
- **Contrast**: Clear size differentiation between hierarchy levels
- **Line Height**: 1.15-1.5 for body text. Tighten for larger text.
- **Letter Spacing**: Slight tracking for headers (+0.5-1pt). Normal for body.

### 2.4 Font Selection Principles
- **Industry Standards**: Finance/law prefer serif. Tech/startup often sans-serif.
- **Reader Accessibility**: 10pt minimum for any text recruiters will read
- **System Fonts**: Ensure fallback fonts for different systems
- **ats-Friendly**: OpenType fonts preferred. Avoid custom encoding.

## 3. Color Systems & Palettes (Plural)

### 3.1 Professional Palettes (Plural)
- **Monochrome**: Black only, shades for hierarchy. Most conservative.
- **Dark+Accent**: Black with single accent color for headers/branding
- **Two-Tone**: Dark text with subtle color for visual interest
- **Industry-Specific**: Creative for creative roles, conservative for finance/law

### 3.2 Color Selection
- **Primary Text**: #000000 or near-black (#111, #222) for readability
- **Secondary Text**: Dark gray (#333, #444) for metadata
- **Accent Colors**: Professional blues (#003366, #1a365d), deep reds (#8b0000)
- **Avoid**: Bright/neon colors, heavy color blocks, gradient backgrounds

### 3.3 Dark Mode Considerations
- **Invert Carefully**: Many ATS systems strip color. Don't rely on color alone.
- **Test Grayscale**: Print in B&W before finalizing. Ensure hierarchy remains.
- **Accessibility**: Contrast ratios apply to human readers too.

## 4. Content Templates (Plural)

### 4.1 Header Templates
```
[Name]
[Title] | [Email] | [Phone] | [Location]
[LinkedIn URL] | [Portfolio/Website]
```

### 4.2 Summary Templates
```
Results-driven [role] with [X] years of experience in [industry/skill].
Proven track record of [key achievement]. 
Seeking to leverage [skills] at [target company type].
```

### 4.3 Experience Entry Templates
```
[Role Title]
[Company Name] | [Location] | [Start Date] – [End Date/Current]
• [Achievement with metrics]
• [Achievement with metrics]  
• [Achievement with impact]
```

### 4.4 Bullet Point Principles
- **Start with Action Verbs**: Led, Developed, Implemented, Reduced, Created
- **Quantify**: Include numbers, percentages, dollar amounts
- **Impact-Focused**: Outcomes, not duties
- **STAR Framework**: Situation, Task, Action, Result
- **ATS Keywords**: Include relevant industry keywords naturally

### 4.5 Skills Formatting
- **Categorized**: Group by type (Technical, Tools, Soft Skills)
- **Comma/Slash Separated**: Standard separators for ATS parsing
- **Relevant First**: Lead with most relevant skills for target role

## 5. ATS Optimization (Plurals)

### 5.1 Document Structure Requirements
- **Standard Section Headers**: Use conventional headers exactly:
  - "Work Experience" or "Professional Experience"
  - "Education" (not "Academic Background")
  - "Skills" (not "Competencies" or "Expertise")
- **Standard Date Formats**: MM/YYYY or Month YYYY. Be consistent.
- **Standard Job Titles**: Match actual titles, avoid creative rewrites

### 5.2 Formatting Rules (Plural)
- **No Columns**: Some ATS cannot read columns. Use single column.
- **No Text Boxes**: Never use text boxes or tables for content
- **No Headers/Footers**: Some ATS strip them. Put info in body.
- **No Graphics/Icons**: ATS cannot read them. Skip profile images.
- **No Multiple Pages for Same Content**: Duplicate info confuses parsing.

### 5.3 Keyword Optimization
- **Natural Integration**: Keywords in context, not just lists
- **Skill Variations**: Include synonym variations (not just "Python" but also "Python programming")
- **Industry Terms**: Use standard industry terminology
- **Tools/Technologies**: List specific tools by name

### 5.4 File Format Best Practices
- **PDF Preferred**: Most reliable. Preserves formatting.
- **DOCX Backup**: If ATS requires. Watch for formatting loss.
- **Plain Text**: Last resort. Loses all formatting.
- **Name Files**: [FirstName_LastName_Resume.pdf]

## 6. Theme Variants (Plural)

### 6.1 Professional Theme
- **Style**: Conservative, traditional, trustworthy
- **Fonts**: Times New Roman, Georgia, or Calibri
- **Colors**: Black, navy, dark gray
- **Best For**: Finance, law, consulting, corporate roles

### 6.2 Modern Theme
- **Style**: Clean, contemporary, approachable
- **Fonts**: Sans-serif throughout (Arial, Roboto, Calibri)
- **Colors**: Black with subtle accent
- **Best For**: Tech, startups, marketing, creative roles

### 6.3 Technical Theme
- **Style**: Minimal, information-dense, scannable
- **Features**: More skills visibility, project details
- **Structure**: Skills early in document
- **Best For**: Engineering, developer, IT roles

### 6.4 Academic Theme
- **Style**: Comprehensive, publication-focused
- **Features**: Education prominently, publications, research
- **Structure**: Education before experience
- **Best For**: Academic, research, scientific roles

### 6.5 Creative Theme
- **Style**: Distinctive, portfolio-showcasing
- **Features**: Space for media/links, project showcases
- **Consider ATS Limitations**: May need separate ATS version
- **Best For**: Design, marketing, media roles

### 7.1 CSS/Print Styles
- **@page Rules**: Set margins, page size
- **page-break-before/after**: Control section breaks
- **widows/orphans**: Control paragraph breaks
- **print-color-adjust**: Ensure backgrounds print

### 7.2 Responsive Considerations
- **Screen vs Print**: Different CSS for screen vs print
- **PDF Generation**: Account for pagination
- **Multiple Pages**: Handle section breaks across pages

### 7.3 Export Integration
- **Template Variables**: Name, title, email, phone, location, summary, experience[], education[], skills[], projects[]
- **Conditional Rendering**: Show/hide sections based on data
- **Default Values**: Handle missing data gracefully

## 10. Validation Approaches (Plural)

### 10.1 Visual Validation
- **Print Preview**: Always preview in print mode
- **Grayscale Test**: Print B&W, verify hierarchy
- **Zoom Test**: Check at 50%, 100%, 200%
- **Screen Readability**: Test on various screen sizes

### 10.2 ATS Validation
- **Parser Test**: Upload to free ATS parser (Jobscan, Resume.io)
- **Keyword Density**: Check for target role keywords
- **Section Detection**: Verify sections recognized
- **Content Extraction**: Verify all content extracted

### 10.3 Content Validation
- **Spelling Check**: Run spell checker
- **Grammar Check**: Verify grammar
- **Consistency Check**: Dates, formatting, terminology
- **Length Check**: Appropriate length for experience level

## 11. Common Issues & Solutions (Plural)

### 11.1 Formatting Issues
- **Problem**: Text runs off page edge
  - **Solution**: Adjust margins, reduce font size slightly (9pt minimum)
- **Problem**: Uneven spacing
  - **Solution**: Use consistent spacing values, CSS-based layout
- **Problem**: Columns misaligned
  - **Solution**: Avoid columns or use simple two-column only

### 11.2 ATS Issues
- **Problem**: Section not recognized
  - **Solution**: Use exact standard headers
- **Problem**: Content missing in parsed output
  - **Solution**: Remove tables, text boxes, headers/footers
- **Problem**: Keywords not found
  - **Solution**: Add keywords naturally throughout document

### 11.3 Visual Issues
- **Problem**: Looks cluttered
  - **Solution**: Increase white space, reduce content density
- **Problem**: Doesn't stand out
  - **Solution**: Improve header formatting, add subtle accent
- **Problem**: Looks unprofessional
  - **Solution**: Stick to professional fonts/colors
